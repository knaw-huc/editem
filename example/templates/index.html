Workflow

Click the run and kill buttons to start/stop tasks.

The function and script tasks perform the same function, but the function task is coded within the web server, and the script task is coded in a separate python file which is run through a shell.

Each task consists of 10 steps, and issues a progress message after each step. Some progress messages are informational, others are error messages.

All messages appear in real time.

Most steps take 1 second, but steps 3 and 8 take 3 and 8 seconds respectively.

You can kill a task during a long running step, because tasks react to stop signals even if they do not issue progress messages.

Both tasks can run concurrently. Moreover, you can open several browser windows, and see the progress in each of them.

You can start a task in one window and kill it in another. Try clicking random buttons without waiting for tasks to complete; try to refresh one browser window while tasks are running.
